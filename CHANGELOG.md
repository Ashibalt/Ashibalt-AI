# Changelog

Все заметные изменения в расширении **Ashibalt AI** задокументированы здесь.


## [0.5.1] - 2026-02-25

### Fixed

 - Исправлена проблема, из-за которой тратилось больше входных токенов чем есть на самом деле. 

### Added

- **Инструмент `product_check`** — автоматическая QA-проверка веб-страниц без скриншотов и Vision-моделей. Запускает headless-браузер (Chrome/Edge), переходит по указанному URL и выполняет полный аудит страницы:
  - **Viewport overflow** — обнаружение элементов, вызывающих горизонтальный скролл
  - **Sibling overlap** — наложение элементов-соседей в нормальном потоке (баг верстки)
  - **Covered interactives** — кнопки/ссылки, перекрытые другими элементами (недоступные для клика)
  - **Broken images** — изображения, которые не загрузились, + distortion (искажение пропорций)
  - **Dead interactions** — кнопки с `pointer-events: none`, пустые кнопки, неработающие ссылки
  - **Alignment & sizing** — несогласованные размеры/выравнивание в flex/grid-контейнерах
  - **Text clipping** — текст, обрезанный `overflow:hidden` без `text-overflow:ellipsis`
  - **Accessibility** — отсутствующие `alt`, `label`, маленькие тап-таргеты, пропуск уровней заголовков
  - **Console errors** — JavaScript-ошибки при загрузке страницы
  - **Network failures** — HTTP 4xx/5xx ответы и сбои загрузки ресурсов
  
  Поддержка responsive-режима: проверка одновременно на mobile (375px), tablet (768px) и desktop (1440px). Весь вывод — структурированный текст с приоритизированными рекомендациями.

- **Инструмент `ask_user`** — агент может задавать уточняющие вопросы пользователю прямо во время агентского цикла, не прерывая работу. Вызов `ask_user(question, options)` вставляет inline-блок в сообщение ассистента: текст вопроса, 2-3 кнопки-варианта и поле для произвольного ответа. После нажатия «Отправить» агент мгновенно получает ответ и продолжает выполнение задачи.

- **Инструмент `add_commit`** — аналог `git commit` без необходимости устанавливать git. Агент может создать полный файловый бэкап («коммит») любого пути в проекте с именем и произвольным scope (например, `"src/Engine"` или `"."` для всего проекта). Бэкапы хранятся в `~/.Ashibalt/sessions/<id>/commits/` — полные копии файлов + JSON-манифест. Директории `node_modules`, `.git`, `dist` и бинарные файлы пропускаются автоматически.

- **Инструмент `get_commit`** — управление сохранёнными коммитами. Поддерживает четыре действия:
  - `action="list"` — таблица всех коммитов сессии с ID, именем, scope, датой и числом файлов
  - `action="restore"` — восстановление рабочего дерева из выбранного коммита по `commitId`
  - `action="delete"` — удаление коммита для освобождения места
  - `action="diff"` — сравнение снапшота с текущим состоянием файлов (modified / deleted / unchanged)

## [0.4.1] - 2026-02-24

### Added

- **`xray_codebase`** — новый LSP инструмент для быстрого обзора структуры кодовой базы. Выводит дерево файлов с сигнатурами функций/классов, константами и переменными. Поддерживает Python, TypeScript, JavaScript, Go и другие языки.
- **Инструмент `tasks`** — агент может создавать структурированный список задач для отслеживания прогресса. Панель задач отображается в чате с галочками и автоматической очисткой при новом запросе пользователя. Кнопка закрытия панели появляется после завершения всех задач.
- **Интерактивные терминальные промпты** — автоматическое обнаружение y/n, пароля, выбора и других интерактивных запросов в терминале. Показывается UI-диалог с предзаполненным рекомендуемым ответом. Поддержка отправки пустой строки (Enter) для промптов типа «Press Enter to continue».

### Changed

- **Терминальные инструменты объединены** — три инструмента (`run_terminal`, `write_stdin`, `read_background`) объединены в один `terminal` с параметром `action`, что уменьшает нагрузку на контекст модели.
- **Кэш провайдеров персистентный** — кэш автовыбора провайдера OpenRouter теперь сохраняется между перезапусками VS Code через `globalState` с TTL 24 часа. Ранее кэш терялся при каждом перезапуске.
- Улучшена обработка пустых ответов в интерактивном терминале — пустая строка корректно отправляется для промптов, ожидающих нажатия Enter.


## [0.3.1] - 2026-02-23

### Changed

Улучшенная логика автовыбора провайдера OpenRouter (кратко):
- Фильтр: Учитываются только провайдеры с ценой за input_cache_read (поддержка кэша). Если таких нет 
автовыбор отключается для модели на сессию, используется стандартная логика OpenRouter.

- Приоритет 1 (Цена кэша): Сравнивается input_cache_read. Допустимая разница — до $0.03. Отбираются провайдеры с минимальной ценой (включая тех, кто в пределах 3 центов от минимума).

- Приоритет 2 (Цена вывода): Среди отобранных сравнивается completion (output). Допустимая разница — до $0.10. Отсеиваются те, у кого цена выбивается за пределы 10 центов от минимальной.

- Приоритет 3 (Скорость): Из оставшихся выбирается провайдер с наименьшей задержкой (latency_last_30m.p50).
- Фиксация: Выбранный провайдер закрепляется за моделью на всю текущую сессию.

- Обновлена документация (README.ru.md, README.md, README.am.md): актуальное количество инструментов, добавлен providerAutoSelect.ts в архитектуру.

## [0.3.0] - 2026-02-22

### Added

- **LSP Bridge** — семь новых инструментов для прямого взаимодействия с Language Server Protocol через VS Code API:
  - `lsp_definitions` — переход к определению символа
  - `lsp_references` — поиск всех ссылок на символ
  - `lsp_hover` — получение документации (hover info)
  - `lsp_type_definition` — переход к определению типа
  - `lsp_implementations` — поиск реализаций интерфейса/абстрактного класса
  - `lsp_rename` — переименование символа во всём проекте
  - `lsp_code_actions` — получение и применение быстрых исправлений (code actions / quick fixes)
- **Индикатор текущей сессии** — активная сессия выделяется в боковой панели истории.
- **Параметр `case_sensitive`** в инструменте `search` — позволяет выполнять поиск с учётом регистра.
- Исправлена ошибка закрепления окна метрик по клику.
- Исправлена обработка ошибок 404 от провайдеров при маршрутизации запросов.

## [0.2.0] - 2026-02-21

### Added / Changed

- **Автовыбор провайдера** — при отправке запроса расширение автоматически определяет и выбирает наиболее подходящего провайдера для выбранной модели. Работает пока что только с OpenRouter, логика такая: Выбирается и фиксируется на всю сессию провайдер, который поддерживает кеширование и имеет меньшую стоимость. Если цена у кешируемого провайдера выше в 3 раза чем у не кешируемых, то автовыбор отключается. Так же добавлена логка обработки ошибок и сбоев. 
- **Параллельный вызов инструментов (parallel tool calls)** — включён для всех моделей в режиме агента. Модель может вызывать несколько инструментов одновременно в рамках одной итерации, что значительно ускоряет выполнение сложных задач (чтение файлов, запуск команд, поиск и т.д.).

- Оптимизированы файлы webview.
- Интерфейс UI\UX был слегка улучшен.
